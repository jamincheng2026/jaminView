# JaminView 编辑器核心能力重构方案

这份文档用于回答一个核心问题：

- 为什么当前编辑器“已经能用，但核心体验还不够稳”
- 接下来应该如何重构
- 先改什么，后改什么
- 图表、地图、表格、图片这些组件到底该怎么设计

这份文档不追求写成大而空的 PRD，而是给后续开发提供一份明确、可执行的基线。

---

## 1. 当前问题

当前编辑器已经具备：

- 页面级配置
- 组件新增、拖拽、缩放、多选、对齐
- 数据导入
- 数据集绑定
- 组件级手动 JSON
- 预览与发布

但核心问题依然很明显：

### 1.1 图表编辑区像“参数堆”

虽然图表能力已经很多，但右侧面板的问题是：

- 结构层级不够稳定
- 很多配置项放在一起，用户读起来累
- 一部分配置是工程参数，不是产品语言
- 一部分配置仍然带有“预设”思维，和用户自由编辑冲突

### 1.2 页面级配置与组件级配置边界还不够稳定

当前已经有 `页面` 和 `组件` 两个编辑视图，但还需要继续明确：

- 页面级设置负责什么
- 组件级设置负责什么
- 什么应该属于模板默认值
- 什么应该属于用户编辑能力

### 1.3 通用控件规范还不够统一

当前已经开始统一颜色和下拉菜单，但仍需继续收口：

- 颜色选择器
- 透明度
- 边框
- 阴影
- 圆角
- 内边距
- 数值输入
- 选择器/开关

这些控件如果没有统一规范，编辑器会越来越“像功能集合”，而不是一个完整工具。

---

## 2. 重构目标

## 2.1 产品目标

把 JaminView 编辑器从：

- “高保真可交互原型”

推进到：

- “结构稳定的 P0/P1 可用编辑器”

重点不是一口气做完所有高级能力，而是：

- 核心模型稳定
- 面板结构清楚
- 组件能力边界明确
- 继续扩展时不会反复打架

## 2.2 体验目标

重构后的编辑器应该满足：

1. 用户能快速理解右侧当前在改什么
2. 用户不需要在一堆参数里猜组件行为
3. 页面级和组件级职责清楚
4. 图表/地图/表格/图片的编辑体验属于同一产品体系
5. 模板负责默认风格，编辑器负责开放式自定义

---

## 3. 核心原则

## 3.1 模板负责默认，编辑器负责开放

模板可以给出：

- 默认排版
- 默认颜色
- 默认组件组合
- 默认页面结构

但进入编辑器以后，用户应尽量能通过显式参数编辑，而不是继续被“预设”绑住。

所以：

- 主题属于模板层
- 组件编辑层应尽量开放

## 3.2 页面编辑器 + 组件编辑器并存

JaminView 不是纯组件配置器。

编辑器应明确分为两层：

- 页面级
  - 画布尺寸
  - 头部
  - 背景
  - 展示适配
- 组件级
  - 内容
  - 数据
  - 样式
  - 布局

## 3.3 面板分组必须稳定

成熟产品不会把所有配置都直接摊在一个长列表里。

右侧面板的组织必须稳定，不能随着新增字段越来越乱。

---

## 4. 页面级与组件级职责边界

## 4.1 页面级配置

页面级负责整张大屏的全局结构：

- 画布尺寸
- 展示模式
- 背景色 / 渐变 / 背景图
- 头部显隐
- 头部内容
- 时间区
- 状态徽章
- 页面级适配

页面级不负责：

- 单个图表如何配色
- 单个表格如何显示列
- 单个图片如何加遮罩

## 4.2 组件级配置

组件级负责：

- 位置与尺寸
- 内容
- 数据绑定
- 样式
- 组件特有配置

组件级不负责：

- 整张画布背景
- 页面头部
- 全局适配模式

---

## 5. 图表组件模型

图表组件是编辑器核心组件，后续所有图表都应该遵循同一套模型。

一个图表组件应拆成 6 组：

### 5.1 数据

负责：

- 数据来源模式
  - 数据集绑定
  - 手动 JSON
- 数据集选择
- 字段映射
- JSON 校验
- 渲染数据预览

### 5.2 标题

负责：

- 标题文本
- 标题显隐
- 标题颜色
- 标题字号
- 标题字距
- 标题装饰
  - 信号点
  - 分割线
  - 胶囊底
- 标题内边距
- 标题边框
- 标题背景

注意：

- 标题区不再推荐依赖“标题预设”
- 应该由显式参数组合出标题效果

### 5.3 视觉

负责：

- 图表主色
- 辅助色
- 轴文字色
- 网格色
- 卡片背景
- 边框
- 阴影
- 光晕
- 强调色

注意：

- 不再依赖“图表主题包”主导
- 颜色统一走取色器 + 透明度 + 原始值输入

### 5.4 坐标与网格

负责：

- X/Y 轴显隐
- 轴标签显隐
- 标签旋转
- 标签字号
- 轴线显隐
- 网格显隐
- 网格透明度
- 轴线透明度
- 图内留白

### 5.5 标签与图例

负责：

- 数值格式
- 小数位
- 前缀 / 后缀
- 数据标签开关
- 图例开关
- 图例位置
- 图例强度
- 峰值 / 最新值 / 摘要徽章
- 徽章布局

### 5.6 容器

负责：

- 组件背景
- 边框
- 阴影
- 圆角
- 内边距
- 标题与图表内容的空间关系

---

## 6. 右侧面板结构

## 6.1 一级结构

右侧仍保留两层：

- `组件`
- `页面`

规则：

- 一旦选中组件，自动进入 `组件`
- 页面配置不抢占组件视图

## 6.2 图表组件二级结构

图表选中后，右侧面板应组织成：

1. 布局
2. 数据
3. 标题
4. 视觉
5. 坐标与网格
6. 标签与图例
7. 容器

不是把所有字段堆在一个大段里。

## 6.3 默认展开策略

默认建议：

- 数据：展开
- 标题：展开
- 视觉：展开
- 其它：折叠

因为用户最常改的是：

- 数据
- 标题
- 颜色与视觉

---

## 7. 通用控件规范

这是后面所有组件都要复用的，不再允许各自为政。

### 7.1 颜色控件

统一规范：

- 系统取色器
- 透明度输入
- 原始值输入

支持：

- `#RRGGBB`
- `#RRGGBBAA`
- `rgb(...)`
- `rgba(...)`
- `transparent`

不再默认提供一堆颜色预设块作为主入口。

### 7.2 数值控件

统一规范：

- 支持单位展示
  - `px`
  - `%`
  - `deg`
- 内部仍存纯数值或字符串
- 不在多个地方重复造输入框逻辑

### 7.3 选择控件

统一规范：

- 全部用项目内自定义菜单式 `Select`
- 不再使用浏览器原生 `select`

### 7.4 开关控件

统一规范：

- 全部使用同一套 `Switch`
- 同一视觉样式
- 同一标签结构

---

## 8. 组件体系建议

## 8.1 图表组件

首批继续围绕：

- 折线图
- 面积图
- 柱状图
- 饼图
- 指标卡
- 排行列表

## 8.2 表格组件

表格应重点支持：

- 显示列
- 列别名
- 列宽
- 列顺序
- 表头层次
- 数值格式
- 状态色
- 对齐方式
- hover 反馈

## 8.3 图片组件

图片应重点支持：

- 图片地址 / 素材选择
- 填充模式
- 边框
- 阴影
- 遮罩颜色 / 透明度 / 模糊
- 说明区文本
- 说明区背景 / 边框 / 阴影 / 圆角 / 内边距
- 亮度 / 对比 / 饱和 / 灰度

## 8.4 地图组件

地图应重点支持：

- 数据映射
- 点位 / 路线 / 热度
- 色彩控制
- 标签控制
- 辉光与边界
- 3D 轴
- 视角 / 缩放

地图后续仍是编辑器中最重要的高级组件之一。

---

## 9. 分阶段重构顺序

不建议一次性重做整个编辑器，建议分 3 段：

### Phase 1：图表组件面板重构

目标：

- 把图表从“参数堆”改成稳定结构
- 清理图表预设入口
- 标题 / 视觉 / 坐标 / 标签分组清晰

### Phase 2：表格与图片对齐同一套交互

目标：

- 表格和图片的右侧面板结构与图表一致
- 通用控件进一步统一

### Phase 3：地图与页面级能力收口

目标：

- 地图面板也进入稳定结构
- 页面级配置和组件级配置关系完全清楚

---

## 10. 当前重构判断

当前不建议：

- 继续零碎地往右侧面板加新字段
- 继续新增一堆“预设包”
- 继续只在视觉层小修小补

当前最值得做的是：

- 先把图表组件编辑面板重构完成
- 再把表格、图片、地图按同一原则收进来

只有这一步完成后，JaminView 编辑器才有机会从：

- “有很多功能的原型”

走向：

- “核心模型清晰、可长期迭代的产品”
