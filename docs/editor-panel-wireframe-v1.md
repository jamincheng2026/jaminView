# JaminView 编辑器原型线框 V1

这份文档不是需求说明，而是把编辑器主规格文档里的结构直接翻成低保真线框。

用途：

- 给产品确认结构
- 给设计确认分区
- 给开发确认右侧面板到底应该怎么组织

配合文档：

- [editor-master-spec-v1.md](/Users/admin/Documents/jaminview/docs/editor-master-spec-v1.md)

## 1. 编辑器整页结构

```text
+----------------------------------------------------------------------------------------------------------------------+
| 顶部栏                                                                                                               |
| 返回 | 项目名 | 保存状态 | 保存 | 预览 | 发布                                                                        |
+----------------------------------------------------------------------------------------------------------------------+
| 左侧导航 | 左侧资源区                                 | 中间画布舞台                            | 右侧属性面板                |
|---------|---------------------------------------------|-----------------------------------------|-----------------------------|
| 组件    | 当前分类下的组件卡片                         | 画布工具条                               | 顶层：页面 / 组件           |
| 图层    | 当前页面的图层树                             | 缩放 / 吸附 / 网格 / 安全区               | 二层：内容 / 数据 / 样式 / 高级 |
| 数据    | 当前项目数据集与字段                         | 自由排布的大屏画布                        | 当前面板分组内容             |
| 模板    | 当前项目模板入口 / 模板说明                  | 组件选中框 / 对齐线 / 多选框              |                             |
+----------------------------------------------------------------------------------------------------------------------+
```

## 2. 右侧面板顶层结构

### 2.1 无组件选中时

```text
页面
└── 页面
└── 背景
└── 头部
└── 展示
```

### 2.2 选中组件时

```text
组件
├── 内容
├── 数据
├── 样式
└── 高级
```

说明：

- 右侧不再出现图层列表
- 图层永远只在左侧
- 选中组件后默认打开 `组件`
- `高级` 不是所有组件都必须有内容，但结构上要预留

## 9. 当前实现前必须遵守的结构规则

1. 右侧不允许再出现“图层列表”
2. 组件选中后，右侧必须自动切到组件
3. 不同类型组件必须拥有不同面板，而不是共用一坨字段
4. 颜色类输入统一为：
   - 取色器
   - 透明度
   - 原始值输入
5. 图表/地图/表格/图片/文本的面板结构必须先定清楚，再允许继续实现字段
