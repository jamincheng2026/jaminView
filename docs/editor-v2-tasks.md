## 📍 里程碑 1：环境清理与基础设施建设
*此阶段不涉及具体图表业务，重点是架构清洗和 Zustand Store 搭建*

- [x] **Task 1.1: 归档旧代码与依赖重组**
  - 分配：架构/主程
  - 任务：隔离原 `editor-workbench.tsx` 逻辑，拆分出独立的画布和 Sidebar 容器。
- [x] **Task 1.2: Zustand 全局 Store 接入开发**
  - 分配：架构/主程
  - 任务：建立 `editor-store.ts`，实现组件池 `widgets: Widget[]` 的增删改查。
  - 标准：不准存在 Prop Drilling，面板所有输入直接通过 `dispatch` 修改 Store。
- [x] **Task 1.3: Component Registry (组件大管家) 机制开发**
  - 分配：前端成员
  - 任务：基于 `src/packages/registry.ts`，自动抓取 `packages` 下的所有图表并注入左侧组件池资源库。
- [x] **Task 1.4: 统一 UI 组件库 (Editor-UI) 封装**
  - 分配：前端成员
  - 任务：建立 `<CollapseSection>`, `<ToggleSwitch>`, `<NumberStepper>`，严格落实 `#23422a` 和 `#fafaf5` 色值。

## 📍 里程碑 5：高阶赋能大杀器预研与落地（后期阶段）
*基于 GoView 高阶能力的设计，在基础版跑通后提升核心竞争力*

- [x] **Task 5.1: 全局公共数据池 (Data Pond) 体系构建**
  - 分配：架构/后端联调
  - 任务：实现图表剥离自身 API 设置，统一通过 ID 订阅全局接口，支持全局设置时钟轮询（Polling）。
- [x] **Task 5.2: 图表全局滤镜 (Canvas Filters)**
  - 分配：前端研发
  - 任务：通过右侧 `页面设置` 面板控制 `Store.editCanvasConfig.filters`，使大屏支持一键套用赛博色相(Hue)等 CSS 滤镜。
- [x] **Task 5.3: 组件群组打组能力 (Grouping)**
  - 分配：前端研发
  - 任务：实现图表面板的多选 (Shift+Click)，增加 `GroupType`，实现联合拖拉拽打组联动功能。
- [x] **Task 5.4: 高阶 API 与 SQL 发送器构建**
  - 分配：架构/全栈
  - 任务：面板 API 配置不仅支持 GET/POST，支持开启 "SQL 模式" 传送 `sql: string` 报文给后端拦截器。

## 📍 里程碑 6：工作区交付能力与导入导出闭环
*这一阶段聚焦“可交付”和“可迁移”。编辑器不只要能画，还要能把整份工作区稳定导入、导出、复用。*

- [x] **Task 6.1: 工作区 JSON 导出器与顶部工具栏接入**
  - 分配：前端研发
  - 任务：在 V2 顶部工具栏补齐“导出工作区”能力，导出的内容必须包含 `widgets / dataPonds / dataPondSettings / canvas / projectTitle`，可直接用于异机迁移或备份。
- [x] **Task 6.2: 工作区 JSON 导入器与结构校验**
  - 分配：前端研发
  - 任务：补齐“导入工作区”能力，支持从本地 JSON 恢复 V2 画布；导入前必须做结构校验与错误提示，避免把坏数据直接写进 Store。
- [x] **Task 6.3: 导入导出后的保存 / 预览 / 发布联调**
  - 分配：前端研发
  - 任务：导入后的工作区必须能继续保存、预览和发布；同时补齐顶部状态反馈，让用户知道最近一次导入/导出是否成功。

---

## 📍 里程碑 7：数据治理增强与运行态页面收口
*这一阶段聚焦“数据可加工”和“页面可交付”。把主规划里剩余的 JS 数据拦截器和运行态细节一并补齐。*

- [x] **Task 7.1: JS 数据拦截器（Transformer）构建**
  - 分配：架构/全栈
  - 任务：在公共数据池与组件兼容请求配置中加入 `JavaScript Transformer`，允许对接口响应做前端二次加工，再注入图表或自定义组件。
- [x] **Task 7.2: 数据池结果预览与错误反馈面板**
  - 分配：前端研发
  - 任务：在页面态和组件数据面板中补齐运行态结果预览、错误提示、转换后摘要，让联调时能看见“原始结果 -> 转换结果 -> 最终注入”的闭环。
- [x] **Task 7.3: 预览页 / 展示页 / 发布成功页交付体验收口**
  - 分配：前端研发
  - 任务：统一 V2 运行态页面的空状态、摘要信息和发布反馈；让预览页、展示页、发布成功页都能表达当前项目是否已有草稿、已发布版本以及当前画布规模。
