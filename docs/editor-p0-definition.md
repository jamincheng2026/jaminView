# JaminView 编辑页 P0 需求定义

这份文档的目标很简单：

- 把 `编辑页` 第一版真正要做的功能钉死
- 避免继续按设计稿“看起来很全”，但开发时不知道哪些要真做
- 让我们后面开始开发编辑页时，有一份明确的执行边界

这份文档优先级高于“页面上看起来有什么”，低于产品主线本身。

---

## 1. 编辑页在产品里的位置

编辑页不是独立产品，它是这条主线里的核心一步：

`选模板 -> 创建项目 -> 导入数据 -> 编辑页 -> 预览 -> 发布`

所以编辑页必须解决的是：

- 用户把模板变成“自己的页面”
- 用户把导入的数据绑定到页面组件上
- 用户能完成一次可展示的修改

编辑页不需要在第一版解决的是：

- 完整低代码平台
- 全自由设计工具
- 复杂多人协作
- 高级自动化和脚本系统

---

## 2. 一句话定义

`JaminView 编辑页 P0` 是一个基于模板的大屏工作台，用户可以在固定画布上调整组件、绑定静态数据、预览结果并进入发布流程。

---

## 3. 第一版成功标准

如果编辑页第一版做成了，应该满足这 5 个结果：

1. 用户可以打开一个模板项目进入编辑页
2. 用户可以看到完整的大屏草稿画布
3. 用户可以选中组件并修改基础属性
4. 用户可以给图表或表格绑定已导入的数据集
5. 用户可以从编辑页进入预览和发布流程

如果这 5 件事还做不到，说明编辑页还没真正完成 P0。

---

## 4. 第一版产品原则

## 4.1 模板优先

第一版用户不是从空白画布开始，而是基于模板进入编辑页。

所以编辑页第一版的产品心智是：

- `修改`
- `替换`
- `配置`

而不是：

- 从零设计一整张大屏

## 4.2 静态数据优先

第一版编辑页只绑定这些数据：

- JSON
- CSV
- Excel

不做：

- API 实时拉取
- 数据库直连
- WebSocket 实时刷新

## 4.3 视觉结果优先于复杂逻辑

第一版要优先保证：

- 页面结果好看
- 图表能正常显示
- 数据能绑定成功
- 预览和发布闭环通

而不是优先做：

- 高级联动规则
- 条件控制
- 复杂脚本

---

## 5. 页面结构

编辑页采用 `左 - 中 - 右 + 顶部栏` 的固定结构。

```text
+--------------------------------------------------------------------------------------------------+
| 顶部栏: 返回 | 项目名称 | 保存状态 | 预览 | 发布                                                 |
+--------------------------------------------------------------------------------------------------+
| 左侧栏                    | 中间画布区                                       | 右侧属性区        |
|--------------------------------------------------------------------------------------------------|
| 组件库                    | 画布工具条                                        | 基础属性           |
| 图层                      | 画布缩放 / 尺寸 / 网格 / 吸附                     | 样式属性           |
| 数据源                    |                                                   | 数据绑定           |
| 模板                      | 固定大屏画布                                      | 图表配置           |
|                           | 默认 1920 x 1080                                  | 地图配置           |
+--------------------------------------------------------------------------------------------------+
```

第一版不改这个结构。

---

## 6. 顶部栏 P0 需求

## 必做

- 返回项目列表
- 当前项目名称
- 保存状态显示
  - 例如：`已保存 / 保存中 / 未保存`
- 预览按钮
- 发布按钮

## 可以简化实现

- 保存状态可以先做前端 mock 状态，不一定接真实后端
- 项目名可以先只支持展示，不急着在线改名

## 先不做

- 版本历史
- 分享
- 协作成员
- 评论

---

## 7. 左侧栏 P0 需求

第一版左侧栏固定 4 个入口：

1. 组件库
2. 图层
3. 数据源
4. 模板

### 7.1 组件库

作用：

- 往当前画布里新增组件

P0 只需要支持展示和插入以下组件：

- 标题
- 文本
- 数值卡
- 图片
- 装饰块
- 折线图
- 柱状图
- 面积图
- 饼图 / 环图
- 表格
- 地图

### 7.2 图层

作用：

- 查看当前页面上有哪些组件
- 选择当前要编辑的组件

P0 只需要支持：

- 列出组件
- 点击选中
- 基础层级顺序展示

先不做：

- 拖动图层排序
- 图层锁定
- 图层分组

### 7.3 数据源

作用：

- 展示当前项目已导入的数据集
- 给组件绑定数据源时能从这里选择

P0 只需要支持：

- 显示数据集列表
- 显示数据集名称
- 显示基本字段信息

### 7.4 模板

作用：

- 当前项目的模板来源入口
- 帮助用户理解这个页面最初来自哪套模板

P0 可以先很轻：

- 显示模板名称
- 提供“查看模板信息”或“替换模板”的占位入口

先不做：

- 完整模板切换逻辑
- 模板版本管理

---

## 8. 中间画布 P0 需求

## 8.1 画布基础

必须支持：

- 固定设计稿尺寸
- 默认 `1920 x 1080`
- 缩放查看
- 选中组件

建议支持：

- 简单网格
- 吸附辅助

先不做：

- 响应式布局引擎
- 自动排版
- 多画布

## 8.2 组件操作

P0 必须支持：

- 选中组件
- 移动组件
- 调整组件尺寸
- 删除组件

P0 建议支持：

- 复制组件

先不做：

- 多选框选
- 复杂对齐分布工具
- 组件组合 / 解组

## 8.3 初始画布内容

第一版编辑页不是空白页，打开时应该已经有一张模板大屏。

这张大屏至少包含：

- 页面标题
- 数值卡
- 2 到 3 个图表
- 1 个表格
- 1 个地图模块

这样用户进入编辑页后，第一眼就能理解“这是在改一张已有的大屏”。

---

## 9. 右侧属性区 P0 需求

右侧属性区是第一版编辑页最重要的功能区之一。

P0 只保留 5 类内容：

1. 基础属性
2. 样式属性
3. 数据绑定
4. 图表配置
5. 地图配置

## 9.1 基础属性

必须支持：

- `x`
- `y`
- `width`
- `height`
- `zIndex`

## 9.2 样式属性

P0 支持：

- 背景色
- 边框
- 圆角
- 透明度
- 阴影
- 文本颜色
- 字号

## 9.3 数据绑定

P0 支持：

- 选择数据集
- 查看字段列表
- 为组件映射字段

先不做：

- 复杂数据转换
- 自定义公式
- 聚合表达式编辑器

## 9.4 图表配置

P0 支持：

- 图表标题显示开关
- 图例显示开关
- 坐标轴显示开关
- 主色切换
- tooltip 开关

先不做：

- 复杂动画参数
- 自定义 renderer
- 高级交互事件

## 9.5 地图配置

P0 支持：

- 地图类型切换（先很有限）
- 默认视角 / 缩放
- 基础颜色风格
- 点位 / 热区 / 飞线开关

先不做：

- 完整 GIS 图层系统
- 自定义图层脚本
- 复杂空间分析能力

---

## 10. 数据模型 P0 要求

编辑页第一版至少要依赖这些数据结构：

- `Project`
- `ScreenConfig`
- `WidgetInstance`
- `Dataset`
- `ThemeTokenSet`

它们分别解决：

- 当前项目是谁
- 当前画布长什么样
- 页面上有哪些组件
- 组件绑定什么数据
- 当前页面主题和色板是什么

---

## 11. 编辑页与其他页面的关系

编辑页的上游页面：

- 项目列表页
- 选择模板页
- 数据导入页

编辑页的下游页面：

- 预览页
- 发布成功页

所以编辑页必须有两个明确入口：

1. 从“创建项目 / 选模板”进入
2. 从“项目列表已有项目”进入

也必须有两个明确出口：

1. 去预览
2. 去发布

---

## 12. 第一版明确不做

这部分很重要，防止范围继续膨胀。

第一版编辑页不做：

- 空白项目自由搭建模式
- 多画布管理
- 多人协作编辑
- 评论与审批
- 数据库直连
- API 实时拉数
- 自定义脚本逻辑
- 高级联动规则
- 组件市场
- AI 生成布局
- AI 生成图表

---

## 13. 开发顺序建议

不要一上来就把所有编辑页功能一起写完。

正确顺序应该是：

1. 先做编辑器外壳
   - 顶部栏
   - 左侧栏
   - 中间画布
   - 右侧属性区骨架

2. 再做模板大屏初始内容
   - 标题
   - 数值卡
   - 图表
   - 表格
   - 地图容器

3. 再做组件选中和基础属性

4. 再做数据绑定

5. 最后接预览和发布

---

## 14. 最后一句话

编辑页第一版不是要成为“最强编辑器”，而是要让用户在模板基础上，把一张大屏改到可展示、可预览、可发布。

如果后面发现某个功能不服务这件事，那它就不应该进入 P0。
