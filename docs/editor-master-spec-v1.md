## 2. 一句话定义

`JaminView 编辑器` 是一个基于模板的大屏页面编辑器。  
用户通过模板创建项目、导入静态数据、编辑页面与组件、预览并发布为只读展示页。

它不是：

- 通用低代码平台
- 自由设计工具
- BI 分析系统
- 多人协作设计平台

---

## 3. 参考基线

这次编辑器重构，不再以“我们当前已经做了什么”为基线，而以成熟大屏编辑器的产品结构为基线。

### 3.1 参考对象

#### GoView

本次重点借鉴：

- 页面级 / 组件级 / 数据级分层
- 组件基础配置、请求配置、事件配置分层
- 选中什么组件，右侧就只服务当前组件
- 丰富组件，但面板组织稳定

#### DataV

本次重点借鉴：

- `样式 / 数据源 / 高级` 这种稳定的面板组织思路
- 图表类组件按任务分区，而不是参数平铺
- 快速配置与深度配置并存，但优先读得懂

### 3.2 明确不照搬的部分

JaminView 不直接照搬 GoView / DataV 的地方：

- 不照搬它们的具体字段命名
- 不照搬它们的视觉风格
- 不把第一版直接做成“全功能大屏平台”
- 不提前加入复杂联动、事件编排、脚本系统

## 4. 产品边界

### 4.1 本期目标

本期编辑器必须完成一条真实闭环：

`模板 -> 项目 -> 导入数据 -> 编辑页面 -> 预览 -> 发布 -> 展示`

### 4.2 本期成功标准

如果编辑器这期做成了，至少要满足：

1. 用户可以从模板进入编辑器
2. 用户可以编辑整张页面，而不只是改单个组件
3. 用户可以选中组件并编辑内容、数据、样式
4. 用户可以让图表、表格、地图吃到真实数据
5. 用户可以保存、预览、发布
6. 发布后的展示页能稳定读取发布快照

### 4.3 本期不做

明确不做：

- 多人协作
- 评论系统
- 版本历史
- 条件联动编排
- 动画时间轴系统
- AI 生成图表/布局
- 通用脚本系统
- 关系图组件

---

## 5. 核心分层模型

编辑器必须固定为 3 层模型：

### 5.1 页面级

页面级负责整张大屏的全局属性：

- 画布尺寸
- 背景
- 页面头部
- 展示适配
- 页面装饰层

页面级不负责：

- 单个图表配色
- 单个表格列配置
- 单个图片滤镜

### 5.2 组件级

组件级负责当前选中组件的：

- 基础布局
- 内容
- 数据
- 样式
- 组件专属高级项

组件级不负责：

- 页面背景
- 页面头部
- 整体展示适配

### 5.3 数据级

数据级负责：

- 数据集导入
- 数据集编辑
- 数据集绑定
- 字段映射
- 组件级手动 JSON
- 数据预览与校验反馈

数据级不负责：

- 页面布局
- 通用视觉样式

## 6. 编辑器整体布局

编辑器固定使用四区结构：

1. 顶部栏
2. 左侧导航与资源区
3. 中间画布舞台
4. 右侧属性面板

### 6.1 顶部栏

必须包含：

- 返回项目列表
- 项目名称
- 保存按钮
- 保存状态
- 预览
- 发布

### 6.2 左侧区域

左侧必须固定为：

- 组件
- 图层
- 数据
- 模板

图层只允许在左侧出现。  
右侧属性面板不应再出现“图层列表式选择”。

### 6.3 中间画布

中间画布必须支持：

- 自由排布
- 网格吸附
- 安全区
- 缩放
- 多选
- 拖拽
- 拉伸
- 对齐基线
- 批量对齐
- 复制粘贴
- 锁定/解锁

### 6.4 右侧属性面板

右侧面板必须遵守一条硬规则：

> 选中什么组件，就只编辑这个组件；没有选中组件时，才显示页面级配置。

---

## 7. 右侧属性面板总结构

右侧面板统一分为两层：

### 7.1 第一层

- `页面`
- `组件`

### 7.2 第二层

#### 页面模式

固定为：

- 页面
- 背景
- 头部
- 展示

#### 组件模式

固定为：

- 内容
- 数据
- 样式
- 高级

不是所有组件都必须用满 4 个分组，但任何组件都不能退化成“一整个长表单”。

## 8. 页面级能力定义

### 8.1 页面

必须支持：

- 页面标题
- 副标题
- 画布尺寸预设
  - 1920×1080
  - 2560×1080
  - 3840×1080
  - custom

### 8.2 背景

必须支持：

- 纯色
- 渐变
- 背景图
- 背景图透明遮罩
- 背景图填充方式

### 8.3 头部

必须支持：

- 是否显示头部
- 头部样式模板
- 左侧标题信息
- 中间/右侧状态信息
- 时间/日期显示
- 徽章文案

### 8.4 展示

必须支持：

- 完整适配
- 按宽度铺满
- 原始尺寸
- 对齐方式（居中 / 靠上）

## 9. 通用组件基础能力

所有组件都应具备的基础编辑能力：

- X / Y
- 宽 / 高
- 可见性
- 锁定
- zIndex
- 背景色
- 透明度
- 边框颜色
- 边框宽度
- 圆角
- 阴影颜色
- 阴影强度
- 内边距

颜色类配置统一规范：

- 取色器
- 透明度
- 原始颜色值输入

不再使用一排固定预设色块作为主要交互。

## 10. 图表组件规格

### 10.1 范围

本期图表组件包括：

- 指标卡
- 折线图
- 面积图
- 柱状图
- 饼图 / 环图
- 排行列表

### 10.2 面板结构

图表组件右侧固定为：

#### 内容

- 标题文案
- 副说明文案
- 单位前后缀

#### 数据

- 数据来源模式
  - 数据集绑定
  - 手动 JSON
- 数据集选择
- 字段映射
- 手动 JSON 编辑
- JSON 校验
- 实际渲染数据预览

#### 样式

分成 6 组：

1. 标题
2. 配色与视觉
3. 坐标与网格
4. 标签与图例
5. 图形细节
6. 容器

### 10.3 图表样式必须支持

- 标题文本 / 颜色 / 字号 / 字距 / 背景 / 边框 / 圆角 / 内边距 / 信号点 / 分割线
- 系列色板 / 轴文字色 / 网格色 / 图内底色 / 点缀色 / 边框 / 阴影 / 光晕
- X/Y 轴显隐 / 标签显隐 / 标签旋转 / 标签字号 / 轴线显隐 / 网格显隐 / 留白
- 数值格式 / 小数位 / 前缀 / 后缀 / 数据标签 / 图例开关 / 图例位置 / 徽章信息
- 折线平滑 / 线宽 / 线型 / 折点大小 / 面积透明度 / 柱状圆角 / 饼图内环比例

### 10.4 图表需求原则

- 不再以“主题包”为主导
- 模板负责默认视觉
- 用户在编辑器里通过显式参数自由调整
- 所有图表都必须随容器尺寸变化而重排

## 15. 数据系统规格

### 15.1 数据来源模式

所有支持数据的组件，都必须同时支持两种模式：

1. 数据集绑定
2. 手动 JSON

### 15.2 导入能力

本期必须支持：

- CSV
- XLSX / XLS
- JSON

### 15.3 数据编辑

本期必须支持：

- 数据集名称编辑
- 字段名编辑
- 字段类型编辑
- 样例行编辑

### 15.4 数据绑定体验

必须让用户看得清：

- 当前绑定的是哪份数据
- 当前组件实际用了哪些字段
- 手动 JSON 校验结果
- 实际渲染预览

## 16. 需要删除或弱化的旧做法

以下做法应视为需要逐步移除的旧逻辑：

1. 右侧出现图层列表式内容
2. 页面设置和组件设置混在同一堆字段里
3. 颜色只能点预设色
4. 图表主要靠“主题包”切换视觉
5. 不同组件共用一大坨表单
6. 用户一眼看不懂自己在改“内容 / 数据 / 样式”中的哪一层

## 17. P0 / P1 / P2

### 17.1 P0 必做

- 页面级配置稳定
- 图表 / 地图 / 表格 / 图片 / 文本 5 类组件的独立面板结构
- 数据集绑定 + 手动 JSON
- 可视化数据预览
- 拖拽 / 缩放 / 多选 / 对齐 / 复制 / 锁定
- 保存 / 预览 / 发布

### 17.2 P1 强烈建议

- 资源管理
- 图片素材管理
- 表格列顺序拖拽
- 更完整的地图业务样式
- 更完整的展示适配规则

### 17.3 P2 暂不做

- 复杂动画系统
- 联动编排
- 条件控制
- 多人协作
- AI 自动生成
- 关系图组件

## 18. 原型基线

右侧面板目标原型：

```text
页面
└── 页面 / 背景 / 头部 / 展示

组件
├── 内容
├── 数据
├── 样式
└── 高级
```

不同类型组件在这四层下再细分自己的子组，而不是复用同一套字段。

---

## 19. 进入实现前的门槛

只有当下面 4 件事都成立时，才允许继续大规模实现：

1. 当前选中什么组件，右侧面板结构完全明确
2. 页面级和组件级边界不再反复变化
3. 图表 / 地图 / 表格 / 图片 / 文本 的右侧结构都已经定义清楚
4. 颜色、透明度、边框、阴影等通用控件规范已统一

如果这 4 项还没达成，就继续补文档和原型，不要继续堆代码。
