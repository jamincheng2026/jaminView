# JaminView 页面设置补漏说明

这份文档专门补当前编辑器里最容易被忽略的一层能力：

- 页面设置
- 头部展示规则
- 背景配置

之前编辑器更偏“组件编辑器”，用户已经明确指出这会让大屏看起来像写死 demo，而不是可以真正交付的页面编辑器。

---

## 为什么现在必须补这一层

当前编辑器虽然已经支持：

- 组件新增
- 图表和地图配置
- 数据导入
- 预览与发布

但页面级能力还缺得比较明显：

1. 画布头部标题、时间、右上角信息写死  
2. 最终展示是否保留这些信息没有规则  
3. 背景只有固定浅色底，没有页面级背景配置  
4. 用户只能改 widget，不能改整个 screen

这会导致产品体验上出现一个明显断层：

- “组件能改”
- “页面整体却像写死模板”

---

## V1 新增的页面级能力

### 1. 页面头部配置

编辑器需要允许用户配置：

- 页面标题
- 页面副标题
- 是否显示头部
- 是否显示时间与右侧信息
- 是否显示左上角状态徽章
- 徽章文案
- 头部状态文案
- 时间文本
- 日期文本
- 右上角主信息
- 右上角次信息

这意味着：

- 页面头部属于 `screen config`
- 不属于写死模板
- 也不应该硬编码进预览页

### 2. 背景配置

V1 页面背景至少支持：

- 纯色
- 渐变
- 图片

图片背景还要支持：

- 填充方式
  - `cover`
  - `contain`
  - `center`
- 遮罩强度

### 3. 预览页同步

页面设置不是只在编辑器里能看见。

预览页和后续发布页都必须读取同一份页面配置：

- 标题
- 副标题
- 头部显隐
- 时间显隐
- 背景设置

---

## 对产品体验的意义

补上这一层之后，JaminView 的产品心智会更完整：

- 不是只能改 widget 的“组件配置器”
- 而是能配置整张大屏页面的“页面编辑器”

这也更接近 GoView 这类产品真正有价值的地方：

- 页面层
- 组件层
- 数据层

三层同时成立。

---

## V1 之后还可以继续扩的页面级能力

当前这轮先不做，但后面可以继续加：

- 页面尺寸预设
- 页面主题切换
- 背景视频
- 装饰边框开关
- 页面级水印
- 页面级自动刷新时间
- 页面级动态时间组件

---

## 当前实现建议

这一轮开发建议按下面顺序继续：

1. 先把 `screenConfig` 写进草稿模型
2. 编辑器右侧增加 `页面设置 / 背景设置`
3. 中间画布头部改为读 `screenConfig`
4. 预览页同步读取同一份配置
5. 后续再扩展到发布页
