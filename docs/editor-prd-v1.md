# vPro Editor PRD V1

## 1. 文档目标

这份文档只讨论 `vPro` 第一版的大屏编辑器，不讨论官网首页，不讨论长期 AI 愿景。

它要解决的问题是：

- 编辑器第一版到底给谁用
- 第一版的核心任务是什么
- 第一版应该有哪些面板和能力
- 图表、地图、表单、数据导入分别做到什么程度
- 哪些需求暂时不做

这份文档是后续出原型图、设计稿、前端实现和后端接口对齐的基础。

---

## 2. 一句话定义

`vPro Editor V1` 是一个面向业务展示、客户演示和交付场景的模板化大屏编辑器，用户可以基于模板导入静态数据，调整图表和布局，并发布成只读展示页面。

---

## 3. 目标用户

第一版优先服务以下 3 类人：

### 3.1 业务运营 / 数据分析同学

需求：

- 快速做出可展示的大屏
- 不希望从空白开始
- 不想写代码
- 可以接受“模板起步 + 轻量调整”

### 3.2 产品经理 / 方案同学

需求：

- 给客户或内部团队快速做原型
- 演示数据看板和业务驾驶舱方案
- 快速生成可交付页面

### 3.3 前端 / 交付工程师

需求：

- 降低重复搭建成本
- 基于模板快速交付项目演示版本
- 在正式开发前做方案验证

---

## 4. 第一版目标

第一版的核心不是“做最强编辑器”，而是完成这个闭环：

`选模板 -> 创建项目 -> 导入静态数据 -> 调整图表与布局 -> 预览 -> 发布 -> 展示`

换句话说，第一版的成功标准是：

- 用户不是从空白开始
- 用户可以在 10 分钟内出一张能看的大屏
- 发布后能稳定展示

---

## 5. 第一版产品原则

### 5.1 模板优先，不做空白自由设计工具

第一版的产品心智应该是：

- 基于模板快速改

而不是：

- 从零开始做一个无限自由的大屏设计器

### 5.2 静态数据优先

第一版只支持：

- JSON
- CSV
- Excel

不支持：

- 数据库直连
- API 实时拉取
- WebSocket

### 5.3 发布优先于协作

第一版更关注：

- 做出来
- 发布出去

不关注：

- 多人协作
- 评论
- 审批流

### 5.4 视觉质量优先于高级逻辑

第一版更应该先把这些做好：

- 页面结构
- 组件容器
- 图表视觉
- 地图效果
- 发布展示质感

而不是提前做：

- 复杂联动规则
- 脚本逻辑
- AI 生成

---

## 6. 编辑器页面结构

编辑器建议采用经典的 `左 - 中 - 右` 工作台结构。

```text
+--------------------------------------------------------------------------------------------------+
| 顶部栏: 返回项目 | 项目名称 | 保存状态 | 预览 | 发布                                               |
+--------------------------------------------------------------------------------------------------+
| 左侧工作区                | 中间画布区                                         | 右侧属性区        |
|--------------------------------------------------------------------------------------------------|
| 组件库                    | 顶部画布工具条                                      | 组件属性           |
| 图层                      | 画布尺寸切换 / 缩放 / 吸附 / 网格 / 预览标识         | 样式配置           |
| 数据源                    |                                                    | 数据绑定           |
| 模板                      |                固定画布编辑区                       | 图表配置           |
|                           |                默认 1920 x 1080                    | 地图配置           |
|                           |                                                    | 说明 / 帮助        |
+--------------------------------------------------------------------------------------------------+
```

---

## 7. 页面模块定义

## 7.1 顶部栏

必须包含：

- 返回项目列表
- 当前项目名
- 自动保存状态
- 预览按钮
- 发布按钮

可后置：

- 分享
- 版本记录
- 协作入口

## 7.2 左侧栏

第一版建议固定 4 个面板入口：

- 组件库
- 图层
- 数据源
- 模板

说明：

- 不建议第一版把面板做太多
- 左侧栏的目标是让用户快速找到“加什么”和“改什么”

## 7.3 中间画布区

第一版要求：

- 固定尺寸画布
- 默认 `1920 x 1080`
- 支持缩放查看
- 支持基础拖拽与缩放
- 支持选中态
- 支持简单网格或对齐辅助

不要求第一版完成：

- 自由响应式布局引擎
- 复杂自动排版

## 7.4 右侧属性区

第一版建议按 4 类配置组织：

- 基础属性
- 样式属性
- 数据绑定
- 图表 / 地图配置

不建议右侧一上来就做成非常复杂的全量配置中心。

---

## 8. 组件范围

## 8.1 P0 组件

第一版建议只做这批组件：

- 页面标题
- 普通文本
- 数值卡
- 图片
- 装饰块
- 折线图
- 柱状图
- 面积图
- 饼图 / 环图
- 表格
- 3D 地图组件

## 8.2 P1 组件

这些可以在第一版后半段加入：

- 雷达图
- 漏斗图
- 仪表盘
- 轮播榜单
- 视频
- 富文本说明

## 8.3 当前不做

- 复杂自定义代码组件
- 可执行脚本组件
- 联动控制器
- AI 对话组件

---

## 9. 图表能力

## 9.1 图表主库

第一版建议：

- 使用 `VChart`

原因：

- 视觉默认更现代
- 更适合高 UI 要求的大屏组件
- React / Next.js 接入自然

## 9.2 图表配置模型

页面配置层不要直接暴露图表库原始 option。

建议内部统一成：

- 图表类型
- 数据集引用
- 字段映射
- 基础视觉样式
- 主题引用

## 9.3 图表第一版必须可配项

- 标题
- 副标题
- 图例显示
- 坐标轴开关
- 主色 / 系列色
- 数据字段映射
- 数值格式基础选项

第一版不要开放太多底层细节，否则配置面板会很乱。

---

## 10. 地图能力

## 10.1 地图类型分层

第一版地图分为两类：

### A. 区域地图

适合：

- 中国地图
- 省市分布
- 区域着色
- 城市散点

### B. 城市级 3D 地图

适合：

- 城市业务分布
- 门店展示
- 飞线
- 热力
- 城市级演示大屏

## 10.2 技术选型

第一版建议：

- `高德 JSAPI 2.0 + Loca`

后续增强预留：

- `AntV L7`

## 10.3 第一版地图组件能力

必须支持：

- 城市 3D 倾斜视角
- 点位数据展示
- 飞线或轨迹效果
- 基础地图样式切换
- 地图容器与图表容器统一视觉语言

当前不做：

- 路径规划
- POI 搜索
- 多地图供应商切换
- GIS 级复杂空间分析

---

## 11. 数据导入与数据模型

## 11.1 第一版支持的导入方式

- JSON
- CSV
- Excel

## 11.2 数据导入流程

1. 打开数据导入弹窗
2. 上传文件
3. 系统解析
4. 展示字段预览与样例数据
5. 命名数据集
6. 确认导入

## 11.3 第一版必须展示的信息

- 文件名
- 数据集名称
- 字段列表
- 数据样例
- 解析失败提示

## 11.4 第一版不做

- 自动字段智能识别推荐
- 多表关联
- SQL 数据源
- 数据转换脚本

---

## 12. 模板系统与创建项目

## 12.1 第一版模板策略

第一版建议提供 `4 - 6` 个模板，覆盖典型场景：

- 综合运营大屏
- 销售分析大屏
- 门店经营大屏
- 城市业务监控大屏
- 项目演示总览大屏

## 12.2 模板应携带的内容

- 默认画布尺寸
- 默认主题
- 默认组件布局
- 默认示例数据
- 模板说明
- 模板预览图

## 12.3 创建项目流程

1. 用户点击新建项目
2. 输入项目名
3. 选择模板
4. 创建成功
5. 跳转编辑器

---

## 13. 主题与样式系统

第一版建议分成三层：

### 13.1 品牌主题

用于官网和登录后壳子

### 13.2 大屏主题

用于画布背景、卡片容器、装饰线框、标题条、图表容器

### 13.3 图表 / 地图色板

用于：

- 图表系列色
- 地图高亮色
- 飞线色
- 热力色
- 数值强调色

第一版至少提供：

- 深色科技主题
- 稳重商务主题
- 高对比展示主题

---

## 14. 发布与展示

## 14.1 第一版发布定义

发布的本质是：

- 将当前草稿冻结成一个展示快照

## 14.2 发布后必须提供

- 展示链接
- 复制链接
- 打开展示页

## 14.3 展示页要求

- 只读
- 不读草稿态
- 按屏幕比例缩放
- 不显示编辑器 UI

---

## 15. 第一版明确不做

以下能力不应写进当前版本承诺：

- AI 生成页面
- AI 改图表
- NL2SQL
- 数据库直连
- 实时流数据
- 多人协作
- 权限审批
- 历史版本回滚 UI
- 复杂联动逻辑

---

## 16. 页面与弹层清单

围绕编辑器本身，第一版至少需要这些界面：

### 正式页面

- 项目列表页
- 编辑器页
- 预览页
- 展示页

### 关键弹层

- 创建项目
- 模板选择
- 数据导入
- 删除确认
- 发布确认
- 发布成功反馈

---

## 17. V1 验收标准

第一版编辑器至少满足这些条件：

1. 用户能从模板创建项目
2. 用户能导入静态数据
3. 用户能添加和调整基础组件
4. 用户能为图表选择数据集和字段
5. 用户能预览并发布
6. 用户能打开发布后的独立展示页

---

## 18. 当前结论

第一版编辑器不应该被定义成：

- 全能低代码平台
- 零代码 BI 平台
- AI 大屏生成器

更合理的定义是：

`一个以模板和视觉交付为核心的业务大屏编辑器 V1`

这个定义更适合现在的产品阶段，也更适合继续出原型和设计稿。
