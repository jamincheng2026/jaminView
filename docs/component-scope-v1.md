# JaminView 组件范围（V1 / V2）

## 目的
这份文档用于锁定 `JaminView` 第一版编辑器的组件范围，避免在实现过程中不断膨胀范围。

当前判断原则：
- 第一版优先做高频、好交付、能快速形成成品感的组件
- 组件必须适合模板化大屏场景
- 配置模型要尽量统一，避免把编辑器右侧属性系统做炸

## V1 必做组件

### 1. 指标类
- 数值卡
- 趋势指标卡

适用场景：
- 经营总览
- KPI 看板
- 业务监控

### 2. 常规图表类
- 折线图
- 柱状图
- 饼图 / 环形图

适用场景：
- 趋势变化
- 对比分析
- 占比分析

### 3. 表格类
- 数据表格
- 事件列表

适用场景：
- 明细信息
- 任务状态
- 事件流

### 4. 地图类
- 业务地图组件
- 区域热力 / 点位 / 标签 / 3D 轴等地图参数

适用场景：
- 城市业务监控
- 区域分布
- 国内大屏展示

说明：
- V1 地图主线按 `L7` 预留能力设计
- 当前可先用 mock 容器和地图参数壳子推进编辑器

### 5. 基础展示类
- 标题文本
- 图片

适用场景：
- 头部标题
- 品牌标识
- 说明性素材

## V1 暂不做组件

### 1. 关系图
- 网络关系图
- 组织结构图
- 拓扑图
- 依赖链路图

原因：
- 不是第一批高频大屏组件
- 数据结构与配置模型复杂
- 会显著增加编辑器右侧面板复杂度
- 对第一版核心链路价值有限

## V2 可扩展组件
- 关系图（G6 方向）
- 漏斗图
- 雷达图
- 词云
- 时间轴
- 富文本块
- 媒体播放器

## 与技术选型的关系
- 常规图表：`VChart`
- 地图：`L7`
- 关系图：V2 再评估 `G6`

## 当前开发建议
编辑器第一版应先围绕以下组件做通：
1. 数值卡
2. 折线图
3. 柱状图
4. 饼图
5. 表格 / 事件列表
6. 地图

先把这批组件做成：
- 能新增
- 能选中
- 能改标题 / 数据源 / 基础样式
- 能预览
- 能保存草稿

然后再继续扩高级组件。
