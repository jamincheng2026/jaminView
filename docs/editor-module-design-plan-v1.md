# JaminView 编辑器模块化设计规划 V1

这份文档不再讲抽象原则，只做一件事：

> 按 GoView 的能力思路，把 JaminView 编辑器拆成一个个可实施、可验收的模块。

每个模块都回答 5 个问题：

1. 这个模块是干嘛的
2. 用户会怎么用
3. 右侧面板应该长什么样
4. 这一期先做到什么程度
5. 验收时看什么

---

## 0. 总体顺序

后续实施不要乱跳，按这个顺序做：

1. 工作台与页面骨架
2. 页面级配置
3. 通用组件基础能力
4. 图表组件
5. 地图组件
6. 表格组件
7. 图片与文本组件
8. 数据系统
9. 请求配置
10. 数据过滤
11. 高级事件编辑
12. 装饰组件与数字翻牌

---

## 1. 工作台与页面骨架

### 模块目标

让用户先能顺着产品主线走通：

`模板 -> 项目 -> 编辑 -> 预览 -> 发布 -> 展示`

### 用户动作

- 从项目列表进入
- 新建项目
- 进入编辑器
- 保存、预览、发布

### 核心界面

- 顶部栏
- 左侧资源区
- 中间画布
- 右侧属性面板

### 本期范围

- 必做
- 不继续扩协作、评论、版本历史

### 验收标准

- 用户能完整进入编辑器
- 保存、预览、发布链路可走通
- 展示页能打开

---

## 2. 页面级配置

### 模块目标

让用户编辑“整张大屏”，而不只是改单个组件。

### 右侧结构

页面模式固定为：

- 页面
- 背景
- 头部
- 展示

### 必做项

#### 页面

- 页面标题
- 副标题
- 画布尺寸预设

#### 背景

- 纯色
- 渐变
- 背景图
- 背景图遮罩
- 填充模式

#### 头部

- 头部显隐
- 时间显隐
- 徽章显隐
- 头部模板

#### 展示

- 完整适配
- 按宽度铺满
- 原始尺寸
- 居中 / 靠上

### 验收标准

- 页面设置与组件设置彻底分层
- 用户不选组件时能编辑页面
- 预览与展示页读取相同页面设置

---

## 3. 通用组件基础能力

### 模块目标

所有组件都具备统一的编辑体验。

### 右侧结构

组件模式固定有：

- 内容
- 数据
- 样式
- 高级

### 必做项

#### 布局

- X / Y
- 宽 / 高
- 锁定
- 可见性
- zIndex

#### 通用样式

- 背景
- 透明度
- 边框
- 圆角
- 阴影
- 内边距

#### 交互

- 拖拽
- 拉伸
- 多选
- 批量对齐
- 分布
- 复制粘贴
- 删除

### 验收标准

- 不同组件共享统一控件语言
- 颜色控件全部一致
- 组件操作在画布上稳定可用

---

## 4. 图表组件

### 模块目标

把图表做成真正的大屏组件，不是参数堆。

### 组件范围

- 折线图
- 面积图
- 柱状图
- 横向柱状图
- 饼图 / 环图
- 排行列表
- 指标卡
- 数字翻牌（第二阶段）

### 右侧结构

#### 内容

- 标题
- 副说明
- 单位前后缀

#### 数据

- 数据集绑定
- 手动 JSON
- 字段映射
- 数据预览

#### 样式

- 标题
- 配色与视觉
- 坐标与网格
- 标签与图例
- 图形细节
- 容器

#### 高级

- 事件能力预留

### 本期必须支持

- 图表随容器尺寸变化而自适应
- 用户自由配色
- 标签/图例/网格/轴线/容器可编辑
- 手动 JSON + 数据集绑定双模式

### 验收标准

- 不同图表右侧逻辑统一，但字段不乱
- 用户能明显感知“内容 / 数据 / 样式”的区别
- 图表不是只靠模板主题才能看起来好看

---

## 5. 地图组件

### 模块目标

地图要成为真正的业务地图组件，而不是氛围图。

### 右侧结构

#### 内容

- 标题
- 说明

#### 数据

- 数据集绑定
- 手动 JSON
- 区域字段
- 热度字段
- 纬度字段
- 经度字段
- 数据预览

#### 样式

- 底图
- 点位
- 路线
- 标签
- 热力
- 容器

#### 高级

- 缩放
- 3D 轴
- 标记点开关

### 本期必须支持

- 地图基础视觉
- 点位
- 路线
- 标签
- 热力表达

### 第二阶段补

- 更复杂底图能力
- 更真实业务图层
- 更深地图交互

### 验收标准

- 地图样式不靠“主题切换”硬撑
- 右侧字段可理解
- 数据映射有反馈

---

## 6. 表格组件

### 模块目标

表格不是普通后台表格，而是大屏滚动表格组件。

### 右侧结构

#### 内容

- 数据说明

#### 数据

- 数据来源
- 列显示/隐藏
- 列别名
- 列顺序
- 列宽
- 数据预览

#### 样式

- 表头
- 行
- 数值
- 状态色
- 容器

### 本期必须支持

- 滚动表格
- 列配置
- 数值格式
- 行样式

### 验收标准

- 表格看起来像大屏组件，不像默认表格
- 列配置真正生效

---

## 7. 图片与文本组件

### 模块目标

图片和文本不能只作为占位块，而要成为页面表达组件。

### 图片右侧结构

- 内容
- 样式
  - 素材显示
  - 滤镜
  - 遮罩
  - 说明层
  - 容器

### 文本右侧结构

- 内容
- 样式

### 本期必须支持

- 图片填充模式
- 滤镜
- 遮罩
- 说明层
- 文本排版

### 验收标准

- 图片和文本不是通用白卡片
- 大屏里有明确表达能力

---

## 8. 数据系统

### 模块目标

把“导入数据”和“组件绑定数据”真正打通。

### 必做项

- CSV / XLSX / JSON 导入
- 数据集名称编辑
- 字段名编辑
- 字段类型编辑
- 样例行编辑
- 数据集绑定
- 手动 JSON
- 校验
- 数据预览

### 第二阶段补

- 数据源分类
- 请求型数据源
- 多数据源管理

### 验收标准

- 用户知道当前组件吃了什么数据
- 手动 JSON 不是工程玩具，而是可用能力

---

## 9. 请求配置

### 模块目标

把 GoView 那种“请求配置”纳入产品模型。

### 本期策略

#### P0

- 不直接开放复杂请求
- 但数据源模型必须预留 `static / request / manual`

#### P1

- 请求地址
- 请求方法
- 刷新间隔
- 参数
- 返回映射

### 验收标准

- 架构已预留，不会未来推翻数据系统

---

## 10. 数据过滤

### 模块目标

把导入的数据变成组件能用的数据，而不是仅仅原样展示。

### P0

- 字段映射
- 列配置
- 基础格式化

### P1

- 条件过滤
- 排序
- Top N
- 聚合
- 截断

### 验收标准

- 数据过滤成为独立能力，不再混在普通字段里

---

## 11. 高级事件编辑

### 模块目标

把 GoView 的高级事件能力纳入路线，而不是以后临时补。

### P0

- 右侧预留高级区域
- 模型允许挂事件配置

### P1

- 点击跳转
- 打开链接
- 跳转展示页
- 简单联动

### P2

- 条件事件链
- 组件间复杂联动

### 验收标准

- 高级配置区存在明确边界
- 不和内容 / 数据 / 样式混在一起

---

## 12. 装饰组件与数字翻牌

### 模块目标

把大屏该有的成品感补上。

### 组件范围

- 边框装饰
- 装饰角标
- 标签条
- 分割线
- 发光装饰
- 数字翻牌

### P1 必做

- 边框装饰
- 装饰组件
- 数字翻牌

### 验收标准

- 页面不只是“图表 + 地图 + 文本”
- 有完整的大屏成品语言

---

## 13. 实施方式

后续不要再按“想到什么做什么”，而要按模块逐个推进。

每完成一个模块，都要做 3 件事：

1. 功能核对
2. 真实交互验证
3. 面板结构复查

---

## 14. 当前明确结论

从现在开始，JaminView 编辑器不是“再修几个已有功能”，而是按 GoView 对标做系统建设。

后续应该一模块一模块做，不再零散补字段。
