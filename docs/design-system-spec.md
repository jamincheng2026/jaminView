# vPro Design System Spec

## 1. 文档目标

这份文档用于定义 `vPro V1` 的设计系统基线，确保后续：

- 原型图有统一视觉方向
- Stitch / Figma 生成稿不跑偏
- 官网、项目列表、编辑器之间有同源关系
- 前端复现时有明确 token 和样式语言

这份文档不是最终的完整设计系统网站，而是当前阶段的 `品牌与 UI 基线规范`。

---

## 2. 当前确认的视觉方向

当前项目已经确认的主方向是：

- `绿色主色`
- `燕麦色背景`

这意味着 `vPro` 的整体气质不是：

- 蓝紫 AI 概念站
- 赛博朋克大屏风
- 纯黑霓虹科技风

而应该是：

- 温和但专业
- 高级但不冷冰冰
- 有科技感，但更偏产品感和品牌感

一句话描述：

`以绿色建立品牌识别，以燕麦色建立高级和耐看底色，再把编辑器做成这套语言的工具化延展。`

---

## 3. 品牌气质关键词

建议统一使用以下关键词做设计判断：

- 专业
- 克制
- 清爽
- 高级
- 可信
- 有秩序
- 数据产品感

应该避免的感觉：

- 过度未来主义
- 过度发光
- 廉价模板站
- 强 AI 海报感
- 视觉噪音太多

---

## 4. 颜色系统

## 4.1 颜色架构

建议拆成四层：

1. 品牌主色
2. 官网背景中性色
3. 编辑器深色中性色
4. 数据可视化色板

## 4.2 品牌主色

品牌绿色建议承担这些角色：

- 主按钮
- 品牌强调
- 链接高亮
- CTA
- 关键图表强调色

建议方向：

- 稳重、偏自然的品牌绿
- 不要荧光绿
- 不要过黄或过脏

推荐语义：

- `brand-primary`
- `brand-primary-hover`
- `brand-primary-soft`
- `brand-primary-foreground`

## 4.3 官网背景中性色

官网主背景建议采用：

- `燕麦色`
- `米白`
- `暖灰白`

推荐作用：

- 页面底色
- Section 背景层次
- 卡片基底
- 弱分割背景

这样做的好处：

- 比纯白更有气质
- 能和绿色主色形成更自然的品牌关系
- 更适合中文产品官网

推荐语义：

- `surface-base`
- `surface-subtle`
- `surface-raised`
- `surface-strong`

## 4.4 编辑器中性色

编辑器建议延续官网的品牌语言，但做成更高密度、更工具化的界面。

推荐结构：

- 外层工作台：浅燕麦色 / 米灰白
- 面板容器：更偏暖灰或浅中性色
- 画布工作区：更深一点的中性底色
- 交互强调：继续使用品牌绿色

可少量引入：

- 深墨绿灰
- 深灰绿
- 低饱和深色分割层

用途：

- 顶部工具栏
- 左右面板分组
- 画布外框
- 属性面板局部层次

原则：

- 和官网是同一品牌，不是两套系统
- 工具感更强，但不必全黑
- 保持克制，不要满屏发光

## 4.5 数据可视化色板

建议单独维护图表和地图色板，不直接复用品牌主色到所有数据系列。

建议至少包含：

- 主绿
- 青绿色
- 湖蓝
- 金沙色
- 珊瑚橙
- 冷灰蓝

要求：

- 系列间区分明显
- 深色 / 浅色背景都可读
- 不要一整页都是绿色图表

---

## 5. 页面级视觉关系

## 5.1 首页

首页应该体现：

- 品牌识别
- 产品感
- 转化感

建议风格：

- 燕麦色主背景
- 绿色 CTA
- 大面积留白
- 用真实产品截图和图表装饰建立科技感

首页不要做成：

- 深色全屏概念站
- 大量赛博发光区块
- 漂浮卡片堆满页面

## 5.2 项目列表页

项目列表页是首页和编辑器之间的过渡。

建议：

- 仍然沿用浅底体系
- 品牌感弱于首页，工具感强于首页
- 卡片、表头和操作区更规整

## 5.3 编辑器

编辑器建议延续官网品牌体系，但转为更规整、更高密度的工作界面。

应该保留的同源关系：

- 绿色作为主要交互强调色
- 燕麦色和暖灰白继续作为基础界面底色
- 卡片圆角、描边、字体层级一致
- 图表 / 地图容器风格统一

可以变化的部分：

- 画布工作区更深
- 分割层次更明显
- 工具面板更规整

---

## 6. 字体系统

中文产品建议使用：

- 标题：有力量但不过度装饰
- 正文：稳重、可读、适合长时间阅读

建议规则：

- Hero 标题用更强的字重
- Section 标题统一节奏
- 正文不要过细
- 表单、表格、属性面板优先可读性

当前不建议：

- 过度艺术化字体
- 头图用太花的 display font

---

## 7. 圆角、边框、阴影

## 7.1 圆角

建议统一成有限的 3 档：

- 小圆角：表单、标签、按钮
- 中圆角：卡片、面板
- 大圆角：Hero 容器、重点模块

避免：

- 每个模块圆角都不一样

## 7.2 边框

建议：

- 浅底页用细边框和低对比描边
- 深色编辑器用更克制的分割线和容器边界

避免：

- 全靠阴影分层
- 所有卡片都没边界

## 7.3 阴影

建议：

- 官网浅底使用柔和阴影
- 编辑器使用轻微投影 + 分割
- 不做厚重浮空玻璃卡片

---

## 8. 动效原则

第一版可以有动效，但要克制。

建议：

- 页面入场淡入和轻微上移
- hover 微反馈
- Hero 局部动效
- 图表 / 地图局部动态装饰

避免：

- 满屏滚动联动
- 长时间强 scrub 动画
- 装饰性大于信息表达

---

## 9. 组件规范方向

第一版需要先统一这些组件视觉：

- Button
- Input
- Select
- Tabs
- Card
- Modal / Dialog
- Table
- Empty State
- Chart Container
- Map Container

### 重点说明

`Chart Container` 和 `Map Container` 不能只把图表直接扔进去。

它们应该具备：

- 统一的容器边界
- 标题区
- 辅助说明区
- 状态区
- 深浅主题适配

---

## 10. Stitch / Figma 出稿要求

后续无论 Stitch 还是 Figma，都建议强制遵守：

- 首页主底色基于燕麦色体系
- 主 CTA 使用品牌绿
- 编辑器延续绿色和燕麦色体系，允许局部更深的工具区与画布区
- 不要蓝紫 AI 风
- 不要玻璃态泛滥
- 不要模板站感的圆角大卡片堆叠

---

## 11. 当前对现有代码的提醒

当前仓库里有一部分旧的深色科技风样式和偏蓝色的视觉残留，这些不应继续作为正式方向。

后续如果要继续复现：

- 首页优先向 `绿色 + 燕麦色` 收敛
- 编辑器在此基础上做工具化延展，而不是完全切到另一套深色品牌
- 旧的蓝青色英雄区和过深背景不作为主方向

---

## 12. 当前结论

如果给团队一句最重要的设计约束，应该是：

`vPro 的正式视觉方向是绿色品牌体系 + 燕麦色官网背景 + 同源的工具化编辑器，而不是蓝紫 AI 科技模板风。`
