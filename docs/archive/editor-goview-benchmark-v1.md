# JaminView 对标 GoView 能力对照文档 V1

这份文档的目标很明确：

- 不再只说“参考 GoView”
- 把 GoView 明确具备、且 JaminView 需要吸收的能力列出来
- 把这些能力拆成：
  - 必须纳入产品能力范围
  - 当前首发必须做
  - 第二阶段补齐
- 作为后续编辑器实现和验收的对照基线

---

## 1. 结论先行

你刚才确认的这些能力，**JaminView 全部都要**。

这里的“全部都要”不是指第一天把所有代码一次性写完，而是指：

1. 它们都必须进入正式产品能力范围
2. 不能再被当成“可有可无”
3. 后续编辑器设计与实现，必须围绕这些能力搭建

所以从现在开始，JaminView 编辑器的目标，不再只是一个“能改几个图表的大屏原型”，而是：

> 一个具备页面控制、组件编辑、数据配置、请求能力、事件能力、主题能力和丰富可视化组件体系的大屏编辑器。

---

## 2. GoView 对标目标

结合公开资料，目前 JaminView 明确要吸收的 GoView 能力分为两大类：

### 2.1 能力层

这些不是单个功能点，而是编辑器的系统层：

1. 工作台
2. 请求配置
3. 数据过滤
4. 高级事件编辑
5. 主题色 / 亮白主题

### 2.2 组件类别

这些是必须纳入 JaminView 编辑器的组件能力池：

1. 柱状图
2. 横向柱状图
3. 折线图
4. 面积图
5. 饼图 / 环图
6. 文字
7. 图片
8. 滚动排名列表
9. 滚动表格
10. 边框装饰
11. 装饰组件
12. 数字翻牌

---

## 3. JaminView 采纳原则

JaminView 对 GoView 的采纳原则不是“照抄界面”，而是：

### 3.1 全量吸收能力层

上面列出的能力层，JaminView 都需要。

### 3.2 重做适合自己的结构

我们不照搬 GoView 的具体字段名和视觉，但要吸收它的产品结构：

- 页面级配置
- 组件级配置
- 数据级配置
- 请求与事件分层
- 丰富组件池

### 3.3 分阶段落地

虽然全部都要，但必须分阶段：

- `P0`: 让编辑器成为真正可用的产品
- `P1`: 把动态数据、事件与装饰体系补齐
- `P2`: 做更高级的联动、动画和企业能力

---

## 4. 能力层对照

## 4.1 工作台

### GoView 侧含义

工作台不是单页，而是一整套编辑上下文：

- 项目
- 页面
- 组件
- 图层
- 数据
- 配置
- 预览
- 发布

### JaminView 必须具备

- 项目列表页
- 模板创建页
- 编辑页
- 预览页
- 发布成功页
- 展示页

### 当前状态

- 已有雏形
- 但编辑页核心体验还不稳

### 结论

- `P0 必做`

---

## 4.2 请求配置

### GoView 侧含义

组件除了静态数据外，还能有请求配置能力：

- 请求地址
- 请求方法
- 刷新间隔
- 参数
- 返回数据映射

### JaminView 应如何采纳

JaminView 第一版不需要直接开放复杂真实 API 接入，但必须把模型留出来。

所以拆成两步：

#### P0

- 静态数据导入
- 数据集绑定
- 手动 JSON
- 数据集预览与编辑

#### P1

- 组件级请求配置
- 页面级数据刷新策略
- 数据源类型区分：
  - 静态
  - 请求
  - 手动

### 结论

- `能力必须纳入范围`
- `P1 做真实请求配置`

---

## 4.3 数据过滤

### GoView 侧含义

不是把数据接进来就结束，而是支持对数据做二次处理：

- 字段筛选
- 条件过滤
- 排序
- 截断
- 聚合
- 映射

### JaminView 应如何采纳

JaminView 第一版至少要支持轻量数据过滤层。

#### P0

- 字段映射
- 列显示隐藏
- 列顺序
- 列宽
- 基础数值格式
- 图表手动 JSON

#### P1

- 过滤条件
- 排序
- Top N
- 聚合方式
- 组件级数据截断

### 结论

- `能力必须纳入范围`
- `P0 做轻量数据整形`
- `P1 做真正数据过滤器`

---

## 4.4 高级事件编辑

### GoView 侧含义

组件不只是静态展示，还能定义：

- 点击事件
- 跳转
- 组件联动
- 显隐控制
- 变量传递

### JaminView 应如何采纳

这块不能在 P0 里做太深，但必须留结构。

#### P0

- 右侧预留 `高级`
- 页面/组件模型允许挂事件配置

#### P1

- 点击跳转
- 打开链接
- 跳转展示页
- 简单联动事件

#### P2

- 条件事件链
- 组件间复杂联动

### 结论

- `能力必须纳入范围`
- `P1 开始落地`

---

## 4.5 主题色 / 亮白主题

### GoView 侧含义

它支持整体主题与明暗切换，不只是单个图表配色。

### JaminView 应如何采纳

JaminView 必须支持：

- 编辑器整体主题体系
- 页面模板默认主题
- 明亮展示主题
- 深色展示主题
- 主题色切换

但注意：

- 模板负责默认主题
- 组件编辑器负责开放式细调

### 结论

- `能力必须纳入范围`
- `P0 做模板默认主题 + 页面主题`
- `P1 做更完整的主题管理`

## 7. 对实施的直接影响

这份对照结论意味着：

1. 右侧编辑面板必须按组件类型完全重构
2. 不能继续共用一大坨表单
3. 请求配置、数据过滤、事件配置必须进入架构预留
4. 组件池要按目标能力扩充，不再只满足当前 demo

---

## 8. 对当前项目的最终判断

JaminView 当前编辑器：

- 还不算 GoView 级别
- 但已经有一部分底座
- 现在最正确的动作不是继续零散补功能，而是：
  - 以这份文档为对照
  - 系统重构编辑器面板
  - 然后按组件池和能力层逐步补齐
